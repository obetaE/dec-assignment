Quick Review: Automatic vs Manual Placement

So far, grid items have automatically filled cells in order. Now you'll learn to place items exactly where you want them.

Automatic Placement

Items flow into cells in source order

.container { display: grid; grid-template-columns: repeat(3, 1fr); }

The Limitation

No control over item positions

Can't create complex layouts

Grid Lines

Invisible numbered lines between columns and rows

Manual Placement

Place items using grid line numbers

.item { grid-column: 2 / 4; grid-row: 1 / 3; }

Precise Control with Grid Lines

Grid lines are the invisible numbered lines that form the coordinate system of your grid. You can place items anywhere in the grid by specifying which lines they start and end at.

Unlike automatic placement where items flow sequentially, manual placement gives you complete control. You can position items in specific cells, make them span multiple rows or columns, and even create overlapping items!

Imagine a parking lot with numbered spaces:

  • Automatic placement: Cars park in the first available space (boring!)
  • Grid lines: Each parking space has lines on all sides with numbers
  • Manual placement: "Park from line 2 to line 4" = takes 2 parking spaces
  • Spanning: "Park from line 1 to line 3 in rows 2 to 4" = a truck taking 4 spaces

Shorthand Properties for Quick Placement

Compact Syntax

CSS Grid offers several shorthand properties to make placement code more concise:

grid-area (Line-based)

Sets all four lines at once: row-start / column-start / row-end / column-end

.item { grid-area: 2 / 1 / 4 / 3; /* row-start / col-start / row-end / col-end */ }

Covers rows 2-3, columns 1-2

grid-area with span

Mix line numbers and span keyword

.item { grid-area: 1 / 2 / span 2 / span 3; /* Start at row1 col2, span 2 rows & 3 cols */ }

Spans rows 1-2, columns 2-4

grid-column & grid-row

The most commonly used shorthand

.item { grid-column: 2 / 4; grid-row: 1 / span 2; }

90% of placement uses this

Pro Tip: Start with grid-column and grid-row shorthands. They're easier to read and maintain than the individual start/end properties or the full grid-area syntax.
